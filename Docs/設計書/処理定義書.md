## 3. 処理詳細

### PRO-001 残高計算
```
残高 = SUM(amount WHERE type='income') - SUM(amount WHERE type='expense')
DBには残高を持たず、常に動的に計算する
```

### PRO-002 家族合計残高（ダッシュボード）
```
家族純資産 = 全childrenの残高合計
先月比増減率 = (今月末残高 - 先月末残高) / 先月末残高 * 100
```

### PRO-003 ダッシュボード棒グラフデータ
```
直近6ヶ月の月次入金合計を集計
JSON: { labels: ["10月","11月",...], data: [50000, 80000, ...] }
```

### PRO-004 目標達成判定
```
達成条件：現在残高 >= goal.target_amount
達成率 = min(floor(残高 / 目標金額 * 100), 100) %
DBに達成フラグは持たない（動的に判定）
```

### PRO-005 子供ラベル判定（学年バッジ）
```
birth_date から年齢を計算し以下のラベルを返す：
0〜2歳  → 乳幼児
3〜5歳  → 幼稚園・保育園
6〜11歳 → 小学生
12〜14歳 → 中学生
15〜17歳 → 高校生
```

### PRO-006 貯蓄推移グラフデータ
```
1. 指定期間（6ヶ月 or 1年）の月ごとの残高を集計
2. 各月末時点の残高 = その月以前の全入出金の合計
3. JSON: { labels: ["12月",...], data: [0, 5000, ...] }
4. Chart.js で折れ線グラフとして描画
```

### PRO-007 履歴月絞り込み
```
1. 年月リストを生成: SELECT DISTINCT DATE_TRUNC('month', transaction_date) ORDER BY DESC
2. 選択中の年月でフィルタリング
3. 日付の降順で表示
```

### PRO-008 新規登録処理
```
1. バリデーション（メール形式・パスワード8文字以上・規約同意）
2. メール重複チェック
3. BCryptでパスワードハッシュ化
4. usersテーブルに role='PARENT' で登録
5. /dashboard へリダイレクト
```

### PRO-009 Googleログイン
```
1. Google OAuth2.0 認証
2. oauth_subject で既存ユーザー検索
3. 一致 → ログイン成功
4. 不一致 → role='PARENT' で新規作成
5. ロールに応じたページへリダイレクト
```

### PRO-010 家族招待
```
1. メール・役割・権限を入力
2. UUIDトークンを生成
3. family_sharesに status='PENDING' で登録
4. 招待URL生成: /family/sharing/accept/{token}
5. 招待受諾時: トークンの invitedEmail とログインユーザーのメールが一致することを確認
   → 不一致の場合は「この招待はあなた宛ではありません」エラー
```

### PRO-011 アクセス制御・所有者チェック
```
未認証           → /login, /register, /family/sharing/accept/*
PARENT           → 全URL（自分が登録した子供のみアクセス可）
PARTNER(EDIT)    → /children/{id}/*, /dashboard, /family
PARTNER(VIEW)    → /children/{id}（読み取りのみ）, /dashboard
GRANDPARENT      → /children/{id}（読み取りのみ）, /dashboard
CHILD            → /children/{自分のid} のみ
他人のchild_id   → IllegalArgumentException（403相当）

【所有者チェック実装箇所】
- TransactionController.detail: PAERENTはfindByIdAndParentで検証
- TransactionController.showNew/create: PAERENTはfindByIdAndParentで検証
- TransactionController.delete: PARENTはchildの所有者確認 + txがchildに属するか確認
- TransactionController.chartData: PAERENTはfindByIdAndParentで検証
- ChildService.findByIdAndParent: childRepository.findByIdAndParentUserIdで所有者確認
- GoalService: findByIdAndChildで所有者確認
- FamilySharingService.revoke/resend: findByIdAndOwnerUserIdで所有者確認
```
